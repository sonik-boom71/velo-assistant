# Велоассистент

Личное приложение для одного велосипедиста: журнал ТО, чек-лист перед выездом,
калькулятор питания/воды и бюджет на апгрейды. Без бэкенда и авторизации — всё
хранится в `localStorage` браузера.

## Стек

- Next.js 14 (App Router) + TypeScript
- Tailwind CSS
- Хранение: один JSON-объект в `localStorage` под ключом `velo-app-data`

## Запуск

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # продакшн-сборка
npm start        # запуск собранного приложения
```

## Структура

- `/` — дашборд: пробег + «+N км», сводки по модулям, экспорт/импорт
- `/maintenance` — журнал обслуживания (износ компонентов, замены, история)
- `/checklist` — чек-лист перед выездом и редактор шаблонов
- `/nutrition` — калькулятор воды и углеводов (без сохранения)
- `/budget` — бюджет на апгрейды

### Код

- `src/lib/types.ts` — модель данных (`AppData`)
- `src/lib/useLocalStorage.ts` — единый SSR-safe хук хранения
- `src/lib/storage.tsx` — контекст `AppData`, дефолты, нормализация импорта
- `src/lib/calc.ts` — расчёты износа и питания, форматирование чисел
- `src/components/WearBar.tsx` — сегментная шкала износа (зелёный → жёлтый → красный)

## Бэкап

Единственный способ сохранить данные вне браузера — кнопки **Экспорт** /
**Импорт** на дашборде (скачивание/загрузка JSON `velo-app-data`).
